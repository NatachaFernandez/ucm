## Versioning Approach

Version updates should reflect meaningful ontology changes.

Examples:

* Major (`v1.x`) — structural or conceptual changes
* Minor (`v0.x`) — new ontology content
* Patch (`v0.x.x`) — documentation or corrections

Workbook filenames remain stable and are not versioned.

Version history is maintained through:

* CHANGELOG
* Git commits
* Repository history
